Frequently Asked Questions

1. The difference between var, let, and const.
var keyword is used to declare variables in function or global scope, it supports reassignment, redeclaring and hoisting.
let keyword is used to declare variables in block scope, it supports updation but not redeclaration.
const keyword is used to declare variables in block scope, it must be inititalized at the beginning and does not support any updation and redeclaration.
2. The difference between map(), forEach(), and filter().
map() is used to get a new array from another array after applying some condition to manipulate the original values, it returns the new array.
forEach() is used to iterate over every element of an array, it returns nothing(undefined).
filter() is used to filter out values following some condition but does not manipulate the original values, it returns the filtered array.
3. Explain arrow functions and how they are different from regular functions.
Arrow functions are declared using var, let or const keyword in contrast to the regular functions which are declared using function keyword. These are used as callback functions. These cannot be used as constructors.
4. How JavaScript Promises work?
JavaScript Promises are a way to handle asynchronous tasks. A pending promise can only result in being either resolved or rejected. It is generally handled using .then(), .catch() and .finally() methods by promise chaining.
5. How closures work in JavaScript?
Closures in JavaScript are those inner functions that retains acess of it's outer function's scope variables even if only the inner function is being executed or the outer function has already executed.
